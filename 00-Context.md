# 00-Context — Стартовый оркестратор

При сбросе контекста:
1. Прочитай этот файл
2. Прочитай `_state.md` — узнай, на каком мы шаге
3. Если онбординг не завершён — следуй `01-Онбординг.md`
4. Если завершён — читай `Профиль атлета.md`, `Профиль тренера.md`, `План тренировок.md`
5. Далее работай по `Анализ тренировок.md`

---

## Структура проекта

```
opencode_sport/
├── 00-Context.md                     <- Стартовый оркестратор (этот файл)
├── 01-Онбординг.md                   <- Пошаговый протокол онбординга (3 сессии)
├── _state.md                         <- Текущий шаг онбординга (обновляется вручную)
│
├── Профиль тренера.md               <- Настройки ИИ: стиль, veto, компетенции
├── Опросник атлета.md               ← Полная анкета для заполнения
├── Профиль атлета.md                ← Заполненный профиль
│
├── Методология.md                   <- Дэниелс, 80/20, норвежский метод
├── Цель и достижимость.md           <- Цель + VDOT-оценка реалистичности
├── План тренировок.md               <- Сгенерированный план
│
├── Локации.md                       <- Места тренировок + покрытие
├── Обувь.md                         <- Кроссовки + распределение
├── Пульсовые зоны.md               <- Зоны ЧСС + протокол теста
├── Блоки ОФП.md                     <- Упражнения, СБУ, растяжка, плио
├── Спортпит.md                      <- Питание и добавки
│
├── Настройка Garmin.md             <- Python + API + .venv
├── Анализ тренировок.md            <- Протокол разбора данных
├── Форматы отчётов.md              <- Шаблоны отчётов
├── garmin/                          <- Синхронизированные данные
│
├── Глоссарий.md                     <- Словарь терминов
├── FAQ.md                           <- Частые возражения
```

## Последовательность действий

| Фаза | Что происходит | Файлы |
|---|---|---|
| **Сессия 1** | Стиль тренера → Профиль атлета → Цель | `01-Онбординг.md` шаги 1-4 |
| **Сессия 2** | Локации → Обувь → Зоны → ОФП | `01-Онбординг.md` шаги 5-8 |
| **Сессия 3** | Garmin → Первый анализ → Калибровка | `01-Онбординг.md` шаги 9-11 |
| **Работа** | Анализ и коррекция тренировок | `Анализ тренировок.md` |

## Правило даты

Перед любым ответом проверяй текущую дату. Сопоставляй с датами в плане. Если дата изменилась — обнови `_state.md` при необходимости.

## Правила верстки .md

- Перед таблицей — пустая строка
- После заголовка — пустая строка
- Ссылки на файлы: `[[Название файла]]`
