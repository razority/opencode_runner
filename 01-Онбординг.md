# 01-Онбординг — Протокол вводных сессий

> Онбординг разбит на 3 сессии, чтобы не перегружать атлета.
> Между сессиями — минимум 1 день на осмысление и сбор данных.

---

## Сессия 1 — Знакомство и база

### Шаг 1. Стиль общения тренера

**Спросить:** «Какой стиль общения тебе удобен?»
- Варианты: экспертный / поддерживающий / жёсткий / с юмором
- Глубина детализации: минимум / средне / максимум цифр
- Как часто напоминать о тренировках: только по запросу / ежедневно / перед каждой

→ Сохранить в `Профиль тренера.md`

### Шаг 2. Опросник атлета

**Спросить:** «Готов ответить на ~20 вопросов о себе? Это нужно, чтобы я понимал твои ограничения и возможности.»

→ Заполнить `Опросник атлета.md` → перенести данные в `Профиль атлета.md`

**Каждый блок предварять пояснением «Зачем это нужно»:**

| Блок | Пояснение |
|---|---|
| Общие данные | Возраст, рост, вес, работа, режим → влияет на восстановление, риск травм |
| Стаж и рекорды | Определяем текущий VDOT, зону ближайшего роста |
| Текущий объём | Недельный километраж, сколько тренировок — база для плана |
| Травмы | Ограничения: какие упражнения нельзя, какие зоны ЧСС под риск |
| Быт | Сон, стресс, алкоголь, работа — корректировка нагрузки |

### Шаг 3. Цель

**Спросить:**
- Какая дистанция?
- Где и когда старт?
- Целевое время?
- Удобные дни для тренировок (сколько дней в неделю готов бегать)
- Есть ли предпочтения по времени суток?

→ Сохранить в `Цель и достижимость.md`

**Оценить реалистичность:**
- Текущий VDOT → целевой VDOT
- Таблица Дэниелса: сколько недель нужно на адаптацию
- Вердикт: зелёный (реально) / жёлтый (надо сдвинуть срок или цель) / красный (пересмотр)

### Шаг 4. Методология

- Объяснить: планирую работать по Дэниелсу + 80/20 + норвежский метод (если подходит)
- Спросить: есть ли предпочтения или ограничения по методологии?

→ Сохранить в `План тренировок.md` (черновик)

## Сессия 2 — Инфраструктура и ОФП

### Шаг 5. Локации

**Спросить:** «Где ты тренируешься? Это важно для привязки типов тренировок к месту.»
- Есть ли рядом стадион с резиновым покрытием? (T-отрезки, СБУ)
- Есть ли парк с ровными дорожками? (лёгкий бег, восстановление)
- Есть ли горки/уклоны? (какой уклон, длина)
- Есть ли длинный ровный участок ~2-3 км? (темповые отрезки)

→ Сохранить в `Локации.md`

### Шаг 6. Обувь

**Спросить:** «Какие кроссовки есть? Распределим по типам тренировок — это продлевает жизнь пене и снижает травмы.»
- Модели, набег, износ
- Есть ли карбоновые/гоночные?
- Нужна ли рекомендация по покупке?

→ Сохранить в `Обувь.md`

### Шаг 7. Пульсовые зоны

**Спросить:** «Знаешь свой пульс ПАНО (порог анаэробного обмена)?»
- Если да — проверяем, как определён, и строим зоны
- Если нет — рекомендую 30-минутный тест Джо Фрила:
  > Разминка 15', затем 30' на максимуме (гонка/темп). Средний пульс последних 20' = ПАНО.
- Если не может провести тест — используем формулу: 180 - возраст (+ поправки)

→ Сохранить в `Пульсовые зоны.md`

### Шаг 8. ОФП / СБУ / растяжка

**Спросить:** «Нужно понять уровень твоей общей физподготовки и слабые места.»
- Каденс (если знает)? Если каденс <170 — добавляем СБУ
- Есть ли опыт ОФП (приседания, выпады, планка)?
- Есть ли ограничения по упражнениям (колени, спина)?
- Делаешь ли растяжку / МФР сейчас?

→ Разработать блоки упражнений → `Блоки ОФП.md`

## Сессия 3 — Garmin и старт работы

### Шаг 9. Настройка Garmin

**Спросить:** «Есть часы Garmin? С ними я смогу анализировать тренировки объективно.»
- Если да → установка Python, .venv, API-ключ
- Если нет → анализ по ручным данным (темп, пульс, дистанция)

→ `Настройка Garmin.md`

### Шаг 10. Первый план

- Свести все данные
- Сгенерировать план по фазам (база → специальный → пик → гонка)
- Согласовать с атлетом

→ `План тренировок.md`

### Шаг 11. Калибровка

- Провести первую неделю по плану
- Разобрать результаты
- Подогнать зоны, темпы, объёмы

→ Переход в рабочий режим по `Анализ тренировок.md`
